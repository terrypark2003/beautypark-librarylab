# 11. 디자인 작업 요청서 (월간 입력물 포맷)

> 업체(라이브러리랩)가 **매월 보내는 엑셀**. 이 표를 바탕으로 그 달의 모든 이벤트 이미지를 제작한다.
> 실물 예시: `assets/event-requests/2026-07-design-request.xlsx` · 파서: `tools/parse_request.py`

## 파일 구조
- 한 통합문서에 **월별 시트**가 누적됨(`YYYY.M` 시트명, 2024.09 ~ 현재). → 과거 시트 = **카피/가격/구성 학습 코퍼스**.
- 각 월 시트가 그 달의 "디자인 작업 요청서".

## 시트 레이아웃 (고정)
| 위치 | 내용 |
|---|---|
| `A1` | 제목 `{M}월 디자인 작업 요청서` |
| `C1` | **정상가** (상시가, 부가세 전) |
| `D1` | **이벤트가** (부가세 전) |
| `E1` | **원내확인용** (이벤트가 부가세 포함) ← 소비자 최종가 기준 |
| `F3/F4` | **플친 와이드** = 캐러셀(3개) |
| `G3/G4` | **플친 리스트** = 캐러셀(5개) |
| `H3` | **인스타(2개)** |
| `I3` | 비고 |
| `A`(병합) | **이벤트 그룹 타이틀** (예: LUCKY 7 FESTA / 바캉스 / Cool한 EVENT / 화수목 / 카톡 플친 전용) |
| `B` | **시술명**(구성·횟수 포함) — **볼드 = 특히 강조되는 대표 상품** |
| 마지막 행 | `월초 강조 : … / 월 중순 강조 : …` (월 테마 카피) |

## 산출물 매핑 (요청서 1장 → 그 달 제작물)
- **플친 와이드** 캐러셀 3컷
- **플친 리스트** 캐러셀 5컷 (보통 이벤트 그룹별 1컷)
- **인스타 게시물 2개** (테마별 카드뉴스)
- (+ 홈페이지 팝업/배너/썸네일, 디스플레이 영상은 [04 디자인 규격](04-design-specs.md) 참고)

## 가격 표기 규칙
- 소비자 노출 최종가 = **이벤트가(VAT 포함, E열)**.
- 정상가 참고 표기 시 동일 기준(VAT 포함)으로 환산(C열 × 1.1) 권장 → 비교가 일관.
- ⚠️ VAT 표기 방식(포함/별도)은 원내 확정 사항.

## 파서 사용
```bash
python3 tools/parse_request.py assets/event-requests/2026-07-design-request.xlsx 2026.7
```
→ 이벤트 그룹/시술명/3종 가격/강조여부/월 테마를 JSON으로 추출(손기재 오류 차단).
생성 데모: `outputs/2026-07/copy-deck.md`
