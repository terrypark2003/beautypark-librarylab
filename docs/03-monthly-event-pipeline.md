# 03. 월간 이벤트 → 배너/웹 게시 파이프라인 (핵심)

> "이벤트를 생성하고, 그 이벤트로 웹사이트 배너를 띄우고, 인스타그램에 홍보한다" — 이 레포의 핵심 업무. 아래는 라이브러리랩이 매월 반복한 표준 흐름이다.

## 단계별 흐름

```
1. 자료 수집/발굴
   └ 신제품·신장비 자료를 원내에 요청 (예: 제품교육자료 PDF)
   └ 강조 희망 시술/장비 협의 (원장 요청 반영)
        ↓
2. 이벤트 기획안 작성 (엑셀)
   └ 익월 이벤트 항목·시술명·정상가·할인가·기간·옵션 정리
   └ 파일명: "{YYMM} 이벤트 기획안 (수정N).xlsx"
        ↓
3. 원내 검수 (가격·시술명·혜택·표기)
   └ 실장(가격/운영) + 원장(시술명/성분/장비 정확성) 분담 검수
   └ 수정사항은 원내에서 한 번에 취합해 전달
        ↓
4. 이벤트 URL 확정
   └ 병원이 홈페이지에 이벤트 페이지 생성 → 링크(URL) 회신
   └ event-introduction?eventIntroductionDurationId=N&eventIntroductionCategoryId=N
        ↓
5. 디자인 1차안 제작
   └ 이벤트 이미지 세트(POP) 제작 → 단톡방 공유 "확인 부탁드립니다"
        ↓
6. 검수 → 수정 → 최종본(POP) 확정
   └ 텍스트/가격/디자인 피드백 반영 (수정 라운드)
        ↓
7. 베리에이션 제작 (확정 POP 기준)
   └ 홈페이지 팝업 / 가로 배너 / 썸네일 / 원내 A4 / 디스플레이 영상
   └ ⚠️ 텍스트 수정 시 영상·베리에이션물 전부 재수정 필요 → 확정 후 진행
        ↓
8. 웹 게시 + SNS 런칭 (매월 1일)
   └ 홈페이지: 팝업/배너/썸네일 등록 (관리자 직접 접속, 잘림 확인)
   └ 인스타: 이벤트 게시물 발행
   └ 카카오: 채널 소식 + 플친 CRM 메시지(이벤트 항목 3~5개 캐러셀)
   └ 메타광고: 이벤트 URL 연결해 집행
        ↓
9. 익월 콘텐츠 일정표 전달
```

## 산출물 체크리스트 (이벤트 1건당)

- [ ] 이벤트 기획안 엑셀 (항목/가격/기간)
- [ ] 인스타 이벤트 게시물 (이미지 세트 + 카피 + 해시태그)
- [ ] 카카오 채널 소식
- [ ] 플친 CRM 메시지 (캐러셀, 이벤트 항목 3~5개)
- [ ] 홈페이지 팝업 이미지 (2400×1350)
- [ ] 홈페이지 가로 배너 (1600×500)
- [ ] 이벤트 썸네일 (1080×540 / 960 가로)
- [ ] 원내 디스플레이 영상 (.3gp / .mp4)
- [ ] (필요 시) 원내 A4 출력물
- [ ] 메타광고 소재 + URL 연결

## 담당/마감 (대체 후 기준)

| 작업 | 마감 | 비고 |
|---|---|---|
| 기획안 1차 | 매월 첫째 주 화 | 아이디어 뱅크([08](08-event-catalog.md)) 활용 |
| 원내 검수 완료 | 매월 12일 | 가격·시술명 확정 |
| 디자인 전체 | 매월 말일 | **앞당겨 25일 목표 권장** |
| 런칭 | 익월 1일(평일) | 인스타+카카오+메타+웹 동시 |

## 카피 표준 구성

1. **후킹 헤드라인** (월 테마 + 시술/혜택)
2. **시술 효과 불릿** (이모지 불릿)
3. **"이런 분께 추천"** 블록
4. **가격 표기**: 정상가 → 할인가 (필요 시 부가세 포함가). 인스타에는 가격 노출 자제(의료광고 리스크), 클릭 유도용 팝업도 가격 미표기 버전 유지.
5. **공통 푸터** ([04-design-specs.md](04-design-specs.md))
